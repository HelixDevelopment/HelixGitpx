# HelixGitpx Administrator Guide

## 1. Introduction

This manual is for organization administrators — the people who own
tenancy, membership, billing, security posture, and policy decisions for
their org on HelixGitpx.

It assumes you have an account and at least one organization you own.

### 1.1 Audience

- Org owners and admins.
- Compliance and security leads who need audit evidence.
- IT administrators rolling HelixGitpx out to a team.

### 1.2 What you'll learn

- **Chapter 2:** creating an organization and inviting members.
- **Chapter 3:** roles, teams, and RBAC.
- **Chapter 4:** SSO / SAML configuration (Scale and Enterprise).
- **Chapter 5:** data residency selection and migration.
- **Chapter 6:** billing and plan management.
- **Chapter 7:** audit logs and evidence export.
- **Chapter 8:** policy overlays for your org.
- **Chapter 9:** provisioning personal access tokens and service accounts.
- **Chapter 10:** incident escalation paths.

### 1.3 Your first checklist

- [ ] Create the organization.
- [ ] Enable MFA as a requirement for members.
- [ ] Set data residency.
- [ ] Invite a secondary owner so you're never the bus factor.
- [ ] Bind your first repository to at least two upstreams.
- [ ] Configure SSO if you have an Identity Provider.
- [ ] Review the default OPA bundle; add any per-org overlay rules.
- [ ] Verify the audit-log export schedule.

### 1.4 Escalation contacts

- Support: `support@helixgitpx.io`.
- Security incidents: `security@helixgitpx.io`.
- Legal: `legal@helixgitpx.io`.
