## 5. Using AI assistance

HelixGitpx ships five AI surfaces: PR summaries, conflict proposals,
label suggestion, code/semantic search, and ChatOps. All five route
through `ai-service` with `NeMo Guardrails` output enforcement.

### 5.1 Configuring models

Org admins pick models per use-case:

- **Local** (Ollama) — ships with Llama-3 or Qwen-2 by default; zero
  data leaves your cluster.
- **On-prem GPU** (vLLM) — for larger models on a dedicated GPU pool.
- **External** (Anthropic, OpenAI) — per-org opt-in; only used if you
  tick *"External providers permitted"* in Settings → AI.

The router is LiteLLM (ADR-0030). Swap models without code changes.

### 5.2 PR summaries

Every PR gets an auto-generated one-paragraph summary when opened. The
summary shows in the PR header and cross-posts to every bound upstream.
Reviewers can ask for a re-summarise at any time.

### 5.3 Conflict proposals

See [Chapter 4](./04-conflicts.md). The AI proposal is advisory; a human
always gates the apply.

### 5.4 Label suggestion

When you open an issue or PR, the AI suggests labels from the org's
defined label set. You accept, reject, or edit — the suggestion never
auto-applies.

### 5.5 Code + semantic search

The **Search** page uses `search-service` which fuses keyword
(Meilisearch), vector (Qdrant), and code regex (Zoekt) via Reciprocal
Rank Fusion. Examples:

- `query: "rate limiter bug"` — semantic + keyword.
- `regex: func \(l \*Limiter\)` — code-only.
- `path: services/adapter-pool file: *.go` — scoped search.

### 5.6 ChatOps

Open **Chat** and type in natural language:

> "open a PR from feat/readme into main, base sha abc123"
> "show me unresolved conflicts for acme/backend"
> "what did bob push yesterday?"

ChatOps turns into calls against the same APIs your CLI uses. Every
command emits an audit event; destructive actions prompt for explicit
confirmation.

### 5.7 Feedback loop

Thumbs up/down on any AI output feeds DPO training (ADR-0032). Your org
opts in to contribute training data; all feedback is PII-scrubbed by
the curator before training.

### 5.8 Killswitch

An org admin can flip `featureflag.ai.enabled=false` at any time — the
AI surfaces disappear, but Git, search, and federation continue. The
killswitch takes effect within 30 seconds.
