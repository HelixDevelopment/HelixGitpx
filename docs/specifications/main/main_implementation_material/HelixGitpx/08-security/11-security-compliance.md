# 11 — Security & Compliance

> **Document purpose**: Provide the **authoritative security architecture**: identity, secrets, policy, supply chain, CI quality gates (SonarQube, Snyk), and the compliance programme (SOC 2, ISO 27001, GDPR, SLSA).

---

## 1. Threat Model (STRIDE)

| Threat | Asset | Attack | Mitigation |
|---|---|---|---|
| **Spoofing** | User account | Stolen token | MFA, short TTLs, device binding, anomaly detection |
| | Service identity | Cert theft | SPIFFE SVIDs rotated hourly |
| | Webhook origin | Forged webhook | HMAC verify, replay window, source IP allowlist |
| **Tampering** | Repos | Malicious push | Required signed commits, branch protection, pre-receive hooks |
| | Container images | Tampered layer | Cosign signatures + Rekor |
| | Audit log | Insider edit | Append-only DB, per-row Merkle anchoring, Rekor mirror |
| **Repudiation** | Actions | Deny having merged | Signed audit, Rekor, trace IDs |
| **Information Disclosure** | Code | Leak through AI | Self-hosted default, explicit opt-in for cloud LLM, PII scrubbing |
| | Secrets | Commit leaks | Gitleaks pre-receive, Snyk DAST, scheduled scans |
| **Denial of Service** | API | Floods | Cloudflare + per-IP/per-token rate limits; circuit breakers |
| | Kafka | Consumer lag | Autoscale, DLQ, backpressure |
| **Elevation of Privilege** | Org roles | Role escalation | OPA policies, policy bundle signing, diff-review gate |
| | K8s | Pod breakout | Distroless images, rootless, seccomp/AppArmor, Kyverno admission |

## 2. Identity & Access

### 2.1 Human Identity

- **OIDC** only. Local password login permitted for dev environments; disabled in prod.
- Pluggable IdPs: Keycloak, Dex, Authentik, Azure AD, Okta, Google Workspace.
- **Passwordless** strongly recommended — FIDO2 / Passkey support.
- **MFA** mandatory for admin roles; strongly encouraged for others (configurable).
- **Session**:
  - Access token: JWT RS256; `aud`, `iss`, `sub`, `scope`; 15 min TTL.
  - Refresh token: opaque, rotating with reuse detection; 30 d TTL.
  - Binding: device fingerprint + IP (soft binding; warn on change).
- **Idle timeout** (default 2 h) → re-auth.
- **Anomaly detection**: ASN / geo / device change heuristics → step-up challenge.

### 2.2 Machine Identity

- **SPIFFE / SPIRE** issues X.509 SVIDs to every workload.
- SVIDs 1 h TTL; rotated in-memory without pod restart.
- All gRPC calls between services require mTLS with mutual SVID validation.
- Kafka: SASL/OAUTHBEARER using SVIDs projected via spire-agent sidecar.
- No long-lived service account tokens on disk — ever.

### 2.3 Authorization

- **Role-Based Access Control (RBAC)** at 4 scopes: global, org, team, repo (+ ref-level protections).
- **Attribute-Based Access Control (ABAC)** overlays via OPA — conditions on request attributes (IP, time, geo, device).
- **Policy-as-Code**: Rego bundles in Git; Argo CD deploys; OPA evaluates.
- Every gRPC request passes through an OPA decision with `{subject, action, resource, context}`.
- Decision latency budget: p95 < 5 ms (cached in-process).

## 3. Secrets Management

- **HashiCorp Vault** is the canonical secrets store.
- Engines:
  - **KV v2** for static secrets (tokens, API keys).
  - **Database engine** for short-lived Postgres credentials (15 min).
  - **Transit engine** for envelope encryption (never expose DEKs outside HSM-backed KMS).
  - **SSH engine** for signed SSH CA → short-lived SSH certs to upstreams.
- Access via **Kubernetes auth method** (SA token → SVID-bound Vault role).
- No secret in env vars at rest — mounted via CSI `secrets-store.csi.k8s.io` and rotated.
- Alternate fallback for small deploys: **Sealed Secrets + SOPS + age** in GitOps.

## 4. Supply Chain Security

### 4.1 Build Integrity

- **SLSA Level 3** at GA; Level 4 for core services by Year 2.
- All builds happen in hermetic runners (self-hosted GitHub Actions on Kata containers or GitLab runners on gVisor).
- Dependencies pinned by hash (`go.sum`, `yarn.lock` integrity, `Gemfile.lock`).
- **Reproducible builds**: deterministic Go builds with `-trimpath`, `-buildvcs=false`, stable timestamps.

### 4.2 Signing & Provenance

- **Cosign keyless** signs every container image against the Sigstore Fulcio CA, recorded in Rekor.
- **in-toto** attestations: build provenance, source integrity, test results, SBOM.
- **SBOM**: CycloneDX 1.5 JSON, generated by `syft`; stored as OCI artefact alongside image.
- **Image admission** (Kyverno): only images with valid Cosign signature + in-toto provenance + SBOM are allowed to run.

### 4.3 Dependency Hygiene

- Nightly `govulncheck` / `npm audit` / Snyk scans.
- Renovate or Dependabot opens PRs; auto-merge patches from allow-listed ecosystems.
- Licence scan (`fossa`) blocks incompatible licences (GPL in non-GPL code).

### 4.4 Commit Signing

- All commits produced by HelixGitpx machinery (bot actions, conflict auto-resolutions) signed via **Gitsign** (Sigstore).
- Users may enforce signed-commit policies on any branch via branch protection.

## 5. CI Quality Gates

### 5.1 SonarQube (Mandatory)

- Self-hosted SonarQube server; each service is a separate project.
- **Quality gate** must pass on every PR:
  - Coverage: ≥ 100 % on new code.
  - Code smells: 0 new.
  - Bugs: 0 new.
  - Vulnerabilities: 0 new.
  - Security hotspots: reviewed.
  - Duplicated lines: ≤ 3 % new.
  - Maintainability: rating A.
  - Reliability: rating A.
  - Security: rating A.
- Merge blocked if gate fails.

### 5.2 Snyk (Mandatory)

- `snyk code` (SAST) on every PR.
- `snyk test` (OSS deps) on every PR.
- `snyk container` on image build.
- `snyk iac` on Helm/Kustomize/Terraform.
- Severity ≥ High blocks merge; Medium triaged within 14 d.

### 5.3 Additional Scanners (Defence in Depth)

| Tool | Scope | Frequency |
|---|---|---|
| **Semgrep** | Custom rules + OWASP | PR |
| **Gitleaks** | Secret detection | PR + pre-receive |
| **Trivy** | Image + FS | PR + nightly |
| **Checkov** | IaC | PR |
| **Terrascan** | Cloud IaC | PR |
| **Kubesec** | K8s manifests | PR |
| **OWASP ZAP** | DAST on staging | Nightly |
| **TruffleHog** | Historical secret scan | Quarterly + post-incident |
| **FOSSA** | Licence compliance | PR |
| **CodeQL** | GitHub Actions SAST | PR + weekly |

### 5.4 Policy (Kyverno) — K8s Admission

- Reject unsigned images.
- Require resource limits.
- Deny `hostPath`, `hostNetwork`, privileged: true.
- Require `runAsNonRoot`, `readOnlyRootFilesystem`.
- Require `NetworkPolicy` in every namespace.
- Require pod anti-affinity for replicasets ≥ 2.
- Enforce SPIFFE trust domain labels.

## 6. Zero-Trust Network

- **Default-deny** NetworkPolicies; every service explicitly lists ingress/egress.
- **Istio Ambient** mTLS at L4 (mTLS + L7 auth via waypoint proxies).
- **Cilium** (CNI) with L7 policies for gRPC paths.
- **Egress gateway**: all outbound traffic leaves through a known IP set (for upstream allowlists).
- **No direct internet** from pods except via egress gateway.

## 7. Data Protection

### 7.1 At Rest

- **Disk encryption**: LUKS on bare metal; cloud provider encryption on managed disks.
- **Database**: TDE (Postgres) + column-level encryption for PII (pgcrypto + Vault transit).
- **Object storage**: SSE with customer-managed keys (MinIO KMS integration).
- **Backups**: encrypted with independent keys; cross-region replica uses different KEK.

### 7.2 In Transit

- **TLS 1.3** minimum; Perfect Forward Secrecy (PFS) ciphers only.
- **HSTS** with `includeSubDomains; preload`.
- **mTLS** internally.

### 7.3 In Use

- **Confidential computing** (optional): AMD SEV-SNP nodes for LLM inference on sensitive orgs (enterprise tier).

### 7.4 Data Residency

- Per-org `region` attribute; data + processing stays in that region.
- Cross-region replication only when explicitly enabled.
- Region-pinned topics in Kafka (topic name prefixed with region).

### 7.5 PII Handling

- Classification (PII / PCI / PHI / confidential) at schema level; columns tagged via `COMMENT ON COLUMN` machine-readable JSON.
- Masking for non-authorized queries (PG `SECURITY DEFINER` views for analytics).
- **Right to erasure** pipeline: `helixctl privacy erase --user-id=…` → GDPR-compliant deletion + tombstones.
- **Data export** (portability): ZIP of user's data (events, repos metadata, messages).

## 8. Logging & Audit

- **Audit log** append-only (see [03-data/04-data-model.md §4.9](../03-data/04-data-model.md)).
- **Immutability** via:
  - Role grants (no UPDATE/DELETE).
  - DB triggers.
  - Periodic Merkle root anchored to Rekor (tamper-evident).
- Every action records: `actor, action, resource, outcome, ip, trace_id`.
- Retention: 1 year hot, 7 years cold.
- Accessible via UI (admins) and `/api/v1/audit` (API).

## 9. Vulnerability Management

- **SIEM**: Wazuh (open-source) ingests logs from hosts, K8s, apps.
- **IDS/IPS**: Falco (runtime threat detection) with custom rules for our process graph.
- **Web Application Firewall (WAF)**: Cloudflare + self-hosted **Coraza** (OWASP ModSecurity CRS).
- **DDoS protection**: Cloudflare at edge; at origin, per-IP connection limits + tarpit.
- **Bug bounty**: HackerOne or self-hosted program, scope and payouts published.
- **Penetration tests**: annual external + quarterly internal; remediation tracked in RBI register.

## 10. Incident Response

- On-call rotation (see [19-operations-runbook.md](../12-operations/19-operations-runbook.md)).
- Severity levels: P1 (customer-impacting outage, data loss) → P4.
- Response SLAs: P1 acknowledge in 15 min, first update in 30 min, status page update.
- **Runbooks** for every known failure.
- **Post-mortems** blameless; published internally, summarised externally for P1/P2.

## 11. Compliance Programme

### 11.1 SOC 2 (Type I in Y1, Type II in Y2)

- Trust Services Criteria: Security (mandatory), Availability, Confidentiality, Privacy.
- Controls documented in `compliance/soc2/controls.md`; evidence collected automatically via Drata / Vanta / manual.

### 11.2 ISO 27001 (Y2)

- ISMS scoped to the HelixGitpx platform.
- Annex A controls mapped to our implementation.

### 11.3 GDPR / CCPA / PIPEDA

- Lawful basis: legitimate interest / contract for primary; consent for AI cloud routing and marketing.
- Data processing register maintained.
- DPIA for LLM workflows.
- DSR endpoints: `export`, `rectify`, `erase`, `object`, `restrict`.
- Vendor list with DPAs tracked.

### 11.4 SLSA

- Level 3 at GA.
- Source integrity: signed tags, protected main.
- Build isolation: hermetic runners, provenance.
- Provenance verification at deploy time.

### 11.5 Licensing

- Code licence: Apache-2.0.
- Third-party licence inventory generated from SBOM.
- Copyleft isolation: no GPL in our monolithic bits; compatible with Apache distribution.
