# M8 — Scale, Harden, GA Design Spec

- **Status:** Approved
- **Date:** 2026-04-20
- **Scope:** Roadmap items 138-161 (Weeks 43-50)

## Goal

Final milestone: take HelixGitpx from feature-complete to General Availability. Stand up a second region, meet performance targets under sustained load, prove DR posture via chaos, pass external security review, hit coverage thresholds, and ship all GA artifacts (docs site, trust center, billing, marketing).

## Deliverables by phase

### 8.1 Multi-region (138-141)
- Second Argo CD Application set (region `eu-west-2`) with Kustomize overlay `impl/helixgitpx-platform/kustomize/overlays/region-b/`.
- `mirrormaker2` Helm chart — replicates Kafka topics A↔B with `.internal` suffix convention.
- `postgres-logical-replication.yaml` (CNPG replica cluster) — read-replica in region-b for repo/org/audit schemas.
- `geodns` chart (CoreDNS + geoip plugin) — health-checked failover between regions.
- `data-residency` ConfigMap surfaced via orgteam-service `SetResidency` RPC (stub OK, wired).

### 8.2 Performance (142-145)
- `tools/perf/` k6 scenarios: `api_baseline.js`, `git_push_pull.js`, `websocket_fanout.js`, `ai_chat.js`.
- `tools/perf/soak-7d.yaml` — Argo Workflow template; schedule disabled (manual trigger).
- `.github/workflows/perf-budgets.yml` — `workflow_dispatch`-only; fails if p99 latencies drift >10%.
- `gpu-autoscaler` values-patch for ollama+vllm (HPA by queue depth).

### 8.3 Chaos & DR (146-148)
- `tools/chaos/` Litmus experiments: broker-kill, pod-evict, partition, disk-full, llm-out, upstream-429-storm (6 YAML manifests).
- `tools/dr/dr-drill-runbook.md` — step-by-step region-loss simulation.
- `docs/operations/runbooks/` — 8 validated runbooks cross-linked from alert rules.

### 8.4 Security hardening (149-152)
- `docs/security/pentest-scope-2026q2.md` — scope document for external firm.
- `docs/security/bug-bounty-program.md` — rules, scope, payouts.
- `docs/security/soc2-type1-evidence-index.md` — maps SOC 2 CC controls to evidence artifacts.
- `docs/security/iso27001-gap-analysis.md` — Annex A control gap table.

### 8.5 Coverage 100% (153-156)
- `tools/coverage-audit/audit.sh` — enumerates Go packages, prints coverage per package, flags <80%.
- `.github/workflows/mutation-testing.yml` — `go-mutesting` per service; `workflow_dispatch`-only.
- `tools/fuzz/` — seed corpora for http, proto, git-smart-http, webhook-hmac parsers.
- `tools/e2e-gaps.md` — audit of untested UI flows, each with ticket reference.

### 8.6 GA (157-161)
- `impl/helixgitpx-docs-site/` — Docusaurus site scaffolded; Helm chart for in-cluster.
- `impl/helixgitpx-web/apps/web/src/app/trust-center/` — new route aggregating SOC 2/ISO/pen-test attestations.
- `billing-service` — new service (Stripe integration stub via LiteLLM-style adapter pattern; Keycloak plan claim).
- `docs/marketing/launch-checklist.md` — public launch go-no-go.
- `RELEASE.md` — GA release notes; `v1.0.0` git tag on repo.

## Architecture

No new platform paradigms. Extends existing patterns:
- Multi-region = two Argo CD projects, Kustomize overlay per region, shared Helm charts.
- Billing-service reuses: Connect-RPC, outbox pattern, OTel, OPA — identical to other M5 services.
- Docs site is a static Docusaurus build deployed as a plain NGINX Pod with Ingress.

## ADRs to add

- **0035** — Two-region active-passive for M8 (vs. active-active later).
- **0036** — MirrorMaker 2 over Confluent Replicator (OSS-first).
- **0037** — CoreDNS + geoip over Route53/Cloudflare (portable across cloud providers).
- **0038** — Stripe over paddle.com for billing (market reach).
- **0039** — Docusaurus over MkDocs for the public site (React ecosystem consistency with the web app).

## Exit criteria

Per roadmap §9: GA-ready; zero P1/P2 open; SLOs green; 30-day beta stable. Represented by `verify-m8-cluster.sh` passing all item checks (138-161).

## Out-of-scope

- SOC 2 Type II (post-GA, year 2).
- ISO 27001 certification itself (M8 only does the gap analysis).
- APAC region deployment (post-GA).
- Active-active multi-region (active-passive first; flip later).
